### 2. 创建用户流程管理器 ✅

**文件位置:** `src/core/user_workflow_manager.py`

**核心功能:**

#### 2.1 流程阶段管理

```python
WorkflowStage:
- NOT_STARTED      # 未开始
- STARTUP          # 启动阶段
- WELCOME          # 欢迎向导
- IDENTITY         # 身份确认
- MAIN_INTERFACE   # 主界面
- EXPERIMENT_SELECTION  # 实验选择
- EXPERIMENT_RUNNING    # 实验进行中
- EXPERIMENT_COMPLETED  # 实验完成
- REVIEW           # 复习回顾
- EXIT             # 退出
```

#### 2.2 用户会话管理

```python
UserSession:
- user_id          # 用户ID
- role             # 用户角色（学生/教师/访客）
- display_name     # 显示名称
- started_at       # 开始时间
- last_active      # 最后活跃时间
- current_stage    # 当前阶段
- experiment_count # 实验次数
- completed_experiments  # 完成数
- total_score      # 总得分
- preferences      # 用户偏好
- state_data       # 状态数据
```

#### 2.3 事件驱动架构

```python
信号（Signals）:
- stage_changed          # 阶段变更
- workflow_event         # 流程事件
- session_started        # 会话开始
- session_ended          # 会话结束
- user_action            # 用户动作
```

**价值:**

- 统一管理用户的完整操作流程
- 提供清晰的状态转换逻辑
- 支持会话恢复和状态保存
- 便于追踪用户行为和问题定位

### 3. 集成流程管理器到主应用 ✅

**修改文件:**

- `main.py` - 在启动时初始化流程管理器
- `src/ui/main_window.py` - 主窗口支持流程管理

**集成点:**

#### 3.1 启动流程集成

```python
# main.py
workflow_manager = get_workflow_manager()
window = MainWindow(container=container, workflow_manager=workflow_manager)

# 连接信号
workflow_manager.stage_changed.connect(window.on_workflow_stage_changed)
workflow_manager.session_started.connect(window.on_session_started)
```

#### 3.2 主窗口响应

```python
# src/ui/main_window.py
def on_workflow_stage_changed(old_stage, new_stage):
    """响应流程阶段变更"""
    - WELCOME → 显示欢迎向导
    - IDENTITY → 确认用户身份
    - MAIN_INTERFACE → 加载主界面
    - 等等...

def on_session_started(session):
    """响应会话开始"""
    - 更新用户ID
    - 更新界面显示
    - 加载用户偏好
```

**价值:**

- 实现流程管理器与UI的解耦
- 支持灵活的流程定制
- 便于扩展和维护

## 五、架构设计

### 5.1 分层架构

```
┌─────────────────────────────────────┐
│          UI 层（用户界面）            │
│  MainWindow, WelcomeWizard, etc.    │
└───────────────┬─────────────────────┘
                │ 信号/槽连接
┌───────────────▼─────────────────────┐
│      流程管理层（业务逻辑）           │
│    UserWorkflowManager              │
│  - 阶段管理                          │
│  - 会话管理                          │
│  - 事件分发                          │
└───────────────┬─────────────────────┘
                │ 数据访问
┌───────────────▼─────────────────────┐
│      数据层（持久化）                 │
│    JSONStore, Files                 │
└─────────────────────────────────────┘
```

### 5.2 事件流

```
用户动作
   ↓
UI 响应
   ↓ (触发)
流程管理器
   ↓ (状态变更)
发送信号
   ↓ (监听)
UI 更新 / 其他模块响应
   ↓ (保存)
数据持久化
```

### 5.3 状态机

```
NOT_STARTED
    ↓ start_workflow()
STARTUP
    ↓ (首次) / (老用户)
WELCOME / IDENTITY
    ↓ complete_welcome_wizard() / confirm_user_identity()
MAIN_INTERFACE
    ↓ start_experiment()
EXPERIMENT_SELECTION
    ↓ 选择实验
EXPERIMENT_RUNNING
    ↓ complete_experiment()
EXPERIMENT_COMPLETED
    ↓ return_to_main_interface()
MAIN_INTERFACE
    ↓ end_session()
EXIT
```

## 八、常见问题

### Q1: 如何跳过欢迎向导？

**方法1（推荐）:** 使用统一启动器，选择 [2] 快速启动

**方法2:** 命令行启动

```bash
python main.py --skip-welcome
```

**方法3:** 删除首次运行标记（不推荐）

```bash
# 删除此文件会导致下次仍显示向导
rm data/.first_run_completed
```

### Q2: 如何重置用户流程状态？

```bash
# 删除会话文件
rm data/last_session.json

# 删除首次运行标记
rm data/.first_run_completed

# 重新启动应用将显示欢迎向导
```

### Q3: 流程管理器支持哪些事件？

```python
# 阶段变更
stage_changed(old_stage, new_stage)

# 流程事件
workflow_event(event)

# 会话管理
session_started(session)
session_ended(session)

# 用户动作
user_action(action_name, data)
```

### Q4: 如何自定义流程？

```python
from src.core.user_workflow_manager import get_workflow_manager

workflow = get_workflow_manager()

# 禁用欢迎向导
workflow.show_welcome_wizard = False

# 启用身份确认
workflow.require_identity_confirmation = True

# 监听自定义事件
workflow.on_event("my_custom_event", my_handler)
```

### Q5: 如何调试流程问题？

```bash
# 1. 运行流程检查工具
python tools/workflow_checker.py

# 2. 查看日志
cat logs/app.log | grep workflow

# 3. 启用详细日志
export LOG_LEVEL=DEBUG
python main.py
```

## 九、技术债务

### 已知限制

1. **简化身份管理**
   - 当前使用简化的student_001模式
   - 教师模式需要进一步开发
   - 多用户管理待实现

2. **会话恢复**
   - 仅支持本地会话恢复
   - 不支持跨设备同步
   - 实验进度恢复有限

3. **错误恢复**
   - 部分异常状态恢复不完整
   - 需要更多的容错处理

### 技术建议

1. **引入状态机库**
   - 使用 `transitions` 或 `python-statemachine`
   - 更严格的状态转换控制

2. **持久化优化**
   - 考虑使用 SQLite 替代 JSON
   - 支持事务和回滚

3. **测试覆盖**
   - 增加流程管理器单元测试
   - 添加集成测试
   - UI自动化测试
