【常用命令】

  列出实验:      实验管理工具.bat list
  查看详情:      实验管理工具.bat info <实验ID>
  添加实验:      实验管理工具.bat add <文件路径>
  验证实验:      实验管理工具.bat validate <实验ID>
  统计信息:      实验管理工具.bat stats
  热重载:        实验管理工具.bat reload
  运行测试:      测试实验编译器.bat

【最简实验模板】

  experiment:
    id: "my_experiment_v1"
    title: "实验标题"
    steps:
      - id: "step_1"
        text: "步骤描述"
        check:
          type: "confirm"

【检查点类型】

  confirm    - 确认操作          用户输入: {"confirmed": true}
  input      - 数值输入          用户输入: {"变量名": 值}
  select     - 选择选项          用户输入: {"变量名": "选项值"}
  sequence   - 依赖检查          检查前置步骤是否完成

【Python API】

  from src.core.enhanced_experiment_manager import EnhancedExperimentManager

  # 创建管理器
  manager = EnhancedExperimentManager(templates_dir="assets/templates")

  # 添加实验
  success, template, msgs = manager.add_experiment("experiment.yaml")

  # 开始会话
  sid, ctrl = manager.start_experiment_session("exp_id", "user_id")

  # 提交步骤
  passed, msg, err = ctrl.submit_step({"confirmed": True})

  # 完成实验
  record = manager.end_experiment_session(sid)

【实验字段】

  必需: id, title, steps
  可选: description, level, duration_min, category, goals,
        reagents, curves, score_rules, version

【难度级别】

  basic          - 基础
  intermediate   - 中级
  advanced       - 高级

【安全级别】

  info      - 信息
  warning   - 警告
  severe    - 严重
  critical  - 危急

【评分表达式变量】

  all_steps_passed      - 所有步骤通过
  total_mistakes        - 总错误数
  completion_rate       - 完成率
  no_safety_warning     - 无安全警告
  no_critical_mistakes  - 无严重错误
  [用户输入变量]        - 如: volume, temperature

【故障排除】

  编译失败:    检查 YAML 语法,确保必需字段存在
  加载失败:    验证模板文件,查看日志
  热重载失败:  确认文件已修改,手动运行 reload
  测试失败:    查看错误信息,检查兼容性
