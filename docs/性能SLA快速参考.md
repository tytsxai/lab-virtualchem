📊 关键SLA指标
═══════════════════════════════════════════════════════════════════════════════

可用性承诺:
  • 月度可用性: ≥99.5%
  • 允许停机: ≤216分钟/月 (3.6小时)
  • 计划维护: 每月第一个周日 02:00-06:00

性能SLO:
  • API P95响应: <200ms
  • API P99响应: <500ms
  • UI渲染时间: <100ms
  • 页面加载时间: <1000ms

容量指标:
  • 并发用户: 100 (目标) | 500 (最大)
  • QPS: 100 (目标) | 500 (最大)
  • 实验/小时: 1000 (目标) | 2000 (峰值)

资源限制:
  • CPU使用: 正常50% | 警告70% | 严重85%
  • 内存使用: 正常256MB | 警告384MB | 严重512MB

🎯 伸缩性策略
═══════════════════════════════════════════════════════════════════════════════

自动伸缩触发:
  • CPU使用率 >70%
  • 内存使用率 >75%
  • 响应时间降级 >20%
  • 队列深度 >500

资源优化:
  • 线程池: 2-10个工作线程
  • 连接池: 2-10个数据库连接
  • 多级缓存: L1(内存) → L2(Redis) → L3(磁盘)
  • 限流: 100 QPS全局, 60 RPM每用户

降级模式:
  • 禁用非核心功能
  • 降低数据保留
  • 请求限流
  • 使用缓存降级
