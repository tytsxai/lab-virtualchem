# 实验系统增强说明

## 概述

本次更新大幅增强了 VirtualChemLab 的实验系统,主要改进包括:

1. ✅ **AI实验编译器** - 支持多种格式的实验创建
2. ✅ **增强的实验管理器** - 动态加载、热重载、版本管理
3. ✅ **错误处理系统** - 完善的错误捕获和恢复机制
4. ✅ **向后兼容性** - 支持旧版本实验模板
5. ✅ **命令行工具** - 方便的实验管理界面

## 新增功能

### 1. AI实验编译器

**文件**: `src/ai/experiment_compiler.py`

**功能**:

- 从YAML/JSON编译实验
- 从字典对象创建实验
- 自然语言转实验(需AI助手)
- 自动格式检测
- 错误验证和修复建议
- 向后兼容旧格式

**使用示例**:

```python
from src.ai.experiment_compiler import compile_experiment

# 从文件编译
result = compile_experiment("experiment.yaml")

# 从字典编译
result = compile_experiment({
    "title": "我的实验",
    "steps": [...]
}, format_type="dict")

# 检查结果
if result.success:
    print(f"成功! 实验ID: {result.template.id}")
    print(f"警告: {result.warnings}")
    print(f"建议: {result.suggestions}")
```

### 2. 增强的实验管理器

**文件**: `src/core/enhanced_experiment_manager.py`

**功能**:

- 实验的增删改查
- 动态加载和热重载
- 会话管理
- 依赖关系检查
- 前置条件验证
- 统计信息

**使用示例**:

```python
from src.core.enhanced_experiment_manager import EnhancedExperimentManager

# 创建管理器
manager = EnhancedExperimentManager(
    templates_dir="assets/templates"
)

# 添加实验
success, template, messages = manager.add_experiment(
    "new_experiment.yaml"
)

# 列出实验
experiments = manager.list_experiments(level="basic")

# 开始实验会话
session_id, controller = manager.start_experiment_session(
    "experiment_id",
    "user_001"
)

# 热重载
count = manager.reload_updated_experiments()
print(f"重新加载了 {count} 个实验")
```

### 3. 实验错误处理器

**文件**: `src/core/experiment_error_handler.py`

**功能**:

- 分类错误管理
- 错误历史记录
- 恢复建议
- 错误统计

**使用示例**:

```python
from src.core.experiment_error_handler import (
    get_error_handler,
    handle_validation_error
)

# 处理错误
error = handle_validation_error(
    "输入值超出范围",
    step_id="step_1"
)

# 查看错误
handler = get_error_handler()
summary = handler.get_error_summary()
print(f"总错误数: {summary['total_errors']}")
```

### 4. 命令行管理工具

**文件**: `tools/experiment_manager_tool.py`

**使用方法**:

```bash
# Windows
实验管理工具.bat <command> [options]

# Linux/Mac
python tools/experiment_manager_tool.py <command> [options]
```

**可用命令**:

```bash
# 列出所有实验
实验管理工具.bat list

# 查看实验详情
实验管理工具.bat info <experiment_id>

# 添加新实验
实验管理工具.bat add <file_path> [--use-ai]

# 更新实验
实验管理工具.bat update <experiment_id> --field <name> --value <val>

# 删除实验
实验管理工具.bat delete <experiment_id> [--force]

# 验证实验
实验管理工具.bat validate <experiment_id>

# 编译实验
实验管理工具.bat compile <source> -o <output>

# 统计信息
实验管理工具.bat stats

# 热重载
实验管理工具.bat reload
```

## 实验模板格式

### 支持的检查点类型

1. **confirm** - 确认操作
2. **input** - 输入数值
3. **select** - 选择选项
4. **sequence** - 依赖关系

详细格式请参考: [实验编译器指南](EXPERIMENT_COMPILER_GUIDE.md)

### 兼容性

系统自动处理以下旧格式字段:

| 旧字段 | 新字段 | 说明 |
|-------|-------|------|
| `difficulty` | `level` | 难度等级 |
| `duration_minutes` | `duration_min` | 预计时长 |
| `instruction` | `text` | 步骤文本 |

## 工作流程

### 添加新实验的完整流程

1. **准备实验内容**
   - 编写YAML/JSON文件
   - 或准备自然语言描述

2. **编译验证**

   ```bash
   实验管理工具.bat compile experiment.yaml -o compiled.yaml
   ```

3. **添加到系统**

   ```bash
   实验管理工具.bat add compiled.yaml
   ```

4. **验证**

   ```bash
   实验管理工具.bat validate <experiment_id>
   ```

5. **测试运行**

   ```python
   manager = EnhancedExperimentManager(...)
   session_id, controller = manager.start_experiment_session(
       experiment_id, user_id
   )
   ```

### 更新现有实验

1. **修改模板文件**
   直接编辑 `assets/templates/<experiment_id>.yaml`

2. **热重载**

   ```bash
   实验管理工具.bat reload
   ```

3. **验证**

   ```bash
   实验管理工具.bat validate <experiment_id>
   ```

## API集成

### REST API支持

增强的实验管理器可以轻松集成到REST API中:

```python
from src.api.server import APIServer
from src.core.enhanced_experiment_manager import EnhancedExperimentManager

# 在API服务器中使用
manager = EnhancedExperimentManager(...)

@app.route('/api/experiments', methods=['POST'])
def add_experiment():
    data = request.json
    success, template, messages = manager.add_experiment(
        data, format_type="dict"
    )
    return jsonify({
        'success': success,
        'experiment_id': template.id if template else None,
        'messages': messages
    })
```

实验管理器支持事件监听:

## 故障排除

### 常见问题

**Q: 编译失败提示"缺少步骤"**

A: 确保实验至少包含一个步骤:

```yaml
steps:
  - id: "step_1"
    text: "步骤描述"
    check:
      type: "confirm"
```

**Q: 热重载不生效**

A: 检查文件修改时间,确保文件确实已更新:

```bash
实验管理工具.bat reload
```

**Q: 兼容性问题**

A: 使用编译器自动转换:

```python
result = compile_experiment(old_format_data)
save_compiled_template(result, "new_format.yaml")
```
