# VirtualChemLab 开发者启动面板

## 概述

开发者启动面板是一个统一的图形化界面，整合了 VirtualChemLab 项目中的所有启动脚本和工具，让开发者可以方便地管理和启动各种功能模块。

## 快速开始

### 启动面板

**方法 1: 使用批处理文件（推荐）**
```bash
开发者面板.bat
```

**方法 2: 直接运行Python脚本**
```bash
python tools/developer_panel.py
```

### 系统要求

- Python 3.10 或更高版本
- tkinter 模块（通常随Python一起安装）
- 所有项目依赖已安装

## 功能模块

### 1. 主应用标签页

#### 标准启动
- **功能**: 启动完整的学生实验模式
- **命令**: `python main.py`
- **适用场景**: 日常开发、测试完整功能

#### 快速启动
- **功能**: 跳过欢迎向导，快速进入实验
- **命令**: `python main.py --skip-welcome`
- **适用场景**: 重复测试、快速调试

#### 许可证验证启动
- **功能**: 启动时验证许可证状态
- **命令**: 调用 `启动带许可证验证.bat`
- **适用场景**: 测试许可证功能、生产环境模拟

### 2. 管理工具标签页

#### 教师控制台
- **功能**: 管理学生实验、查看实验记录
- **脚本**: `tools/teacher_console.py`
- **适用场景**: 教师管理功能开发和测试

#### 管理后台
- **功能**: 系统管理、用户管理、数据统计
- **脚本**: `tools/admin_server_start.py`
- **端口**: 默认 5000
- **访问**: http://127.0.0.1:5000

#### 实验管理工具
- **功能**: 管理实验模板、实验数据
- **脚本**: `tools/experiment_manager_tool.py`
- **适用场景**: 实验内容编辑、模板管理

### 3. 许可证标签页

#### 基础操作
- **获取机器ID**: 生成用于许可证激活的机器标识
- **检查许可证**: 验证当前许可证状态
- **健康检查**: 运行许可证系统健康检查

#### 许可证工具箱
- **功能**: 完整的许可证管理功能集
- **脚本**: `许可证工具箱.bat`
- **包含**: 激活、备份、恢复、导入导出等

#### 备份与恢复
- **备份许可证**: 创建带时间戳的许可证备份
- **列出备份**: 查看所有可用的备份文件
- **自动命名**: `license_backup_YYYYMMDD_HHMMSS.json`

### 4. 开发工具标签页

#### 构建与打包
- **功能**: 使用 PyInstaller 构建可执行文件
- **脚本**: `build.bat`
- **输出**: `dist/VirtualChemLab/`

#### 部署工具
- **功能**: 自动化部署检查和配置
- **脚本**: `deploy.bat`
- **包含**: 依赖安装、目录创建、配置检查等

#### 开发工具箱
- **功能**: 密钥生成、代码检查、格式化等
- **脚本**: `工具箱.bat`
- **工具**: 
  - 生成密钥
  - 检查硬编码
  - 代码格式化（Ruff）
  - 一键验证

#### 测试工具
- **运行所有测试**: 使用 pytest 运行测试套件
- **查看覆盖率**: 显示测试覆盖率报告
- **命令**: `pytest tests/ -v`

### 5. 系统工具标签页

#### 系统维护
- **功能**: 缓存清理、问题诊断、系统修复
- **脚本**: `系统维护工具.bat`
- **包含**:
  - GUI维护工具
  - 清理缓存（全部/过期）
  - 系统诊断
  - 自动修复
  - 健康检查

#### 用户流程检查
- **功能**: 检查用户操作流程的完整性
- **脚本**: `检查用户流程.bat`
- **验证**: 所有用户交互路径

#### 快捷操作
- **清理缓存**: 快速清理所有缓存
- **系统诊断**: 详细的系统问题诊断
- **健康检查**: 快速系统健康状态检查

## 界面功能

### 日志输出区域

实时显示所有命令和进程的输出信息：
- 带时间戳的日志记录
- 自动滚动到最新内容
- 支持复制粘贴

### 底部按钮栏

#### 清空日志
清除日志输出区域的所有内容

#### 停止所有进程
终止所有通过面板启动的进程
- 显示确认对话框
- 安全地终止进程

#### 刷新状态
显示当前运行的进程列表

#### 退出
关闭开发者面板
- 如果有进程运行，会提示确认

### 状态栏

显示当前状态：
- 就绪：无任务运行
- 运行：显示当前任务名称

## 进程管理

### 后台运行
- 所有命令在后台线程中运行
- 不会阻塞界面操作
- 可以同时运行多个任务

### 输出捕获
- 实时捕获标准输出和错误输出
- 统一显示在日志区域
- UTF-8 编码支持中文

### 进程跟踪
- 自动记录运行中的进程
- 支持批量停止
- 进程结束自动清理

## 故障排除

### 面板启动失败

**问题**: 双击后立即关闭
**解决**:
```bash
# 在命令行运行查看错误信息
python tools/developer_panel.py
```

**问题**: 找不到 tkinter
**解决**:
- Windows: 重新安装 Python，确保勾选 tcl/tk
- Linux: `sudo apt-get install python3-tk`
- macOS: 使用 python.org 的官方安装包

### 进程无法启动

**问题**: 点击按钮后没有反应
**检查**:
1. 查看日志区域的错误信息
2. 确认相关脚本文件存在
3. 检查 Python 依赖是否完整

### 中文显示乱码

**问题**: 日志中中文显示为乱码
**解决**:
1. 确保终端编码设置为 UTF-8
2. 检查 Python 文件编码声明
3. 使用 `chcp 65001` 设置代码页

### 权限问题

**问题**: 某些操作提示权限不足
**解决**:
- Windows: 以管理员身份运行
- Linux/macOS: 使用 `sudo` 或调整文件权限

## 配置

### 自定义工具路径

编辑 `tools/developer_panel.py`，修改工具路径：

```python
def run_tool(self, script, name):
    # 自定义工具目录
    script_path = self.custom_tools_dir / script
    ...
```

### 添加新工具

在相应的标签页创建函数中添加：

```python
def create_custom_tab(self, parent):
    frame = ttk.LabelFrame(parent, text="自定义工具", padding="10")
    frame.pack(fill=tk.X, pady=5)
    
    ttk.Button(
        frame,
        text="我的工具",
        command=lambda: self.run_tool("my_tool.py", "我的工具")
    ).pack(fill=tk.X, pady=5)
```

### 修改界面样式

调整窗口大小和主题：

```python
def __init__(self, root):
    # 修改窗口大小
    self.root.geometry("1000x800")
    
    # 修改主题
    style = ttk.Style()
    style.theme_use('vista')  # 可选: 'clam', 'alt', 'default', 'classic'
```

## 技术实现

### 架构设计

```
开发者面板.bat (入口)
    ↓
tools/developer_panel.py (主程序)
    ↓
├── GUI界面 (tkinter)
│   ├── 主应用标签页
│   ├── 管理工具标签页
│   ├── 许可证标签页
│   ├── 开发工具标签页
│   └── 系统工具标签页
│
├── 进程管理
│   ├── 进程启动
│   ├── 输出捕获
│   └── 进程终止
│
└── 工具集成
    ├── Python脚本
    ├── 批处理文件
    └── 命令行工具
```

### 关键技术

- **tkinter**: GUI框架
- **subprocess**: 进程管理
- **threading**: 后台任务执行
- **pathlib**: 跨平台路径处理

### 线程安全

- GUI操作在主线程
- 命令执行在工作线程
- 日志输出线程安全
