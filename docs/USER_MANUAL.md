## 快速开始

### 安装与启动

1. **确保已安装Python 3.8+ (推荐 3.10+)**
   ```bash
   python --version
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **启动程序**
   ```bash
   python main.py
   ```

### 第一次使用

1. 程序启动后会显示主界面
2. 左侧是**实验列表**,右侧是**内容区域**
3. 点击任意实验开始

1. 在左侧列表中浏览可用实验

3. 右侧显示实验详情


### 操作步骤

#### 1. 确认型检查点

- **界面**: 复选框 `☑ 我已确认...`
- **操作**: 勾选后点击"提交"

#### 2. 输入型检查点

- **界面**: 输入框 + 单位标签
  ```
  请输入移取的HCl体积: [_______] mL
  ```
- **操作**: 输入数值后点击"提交"
- **提示**: 程序会检查是否在允许范围内

#### 3. 选择型检查点

**单选**:
```
选择合适的指示剂:
  [ ▼ 请选择 ]
```

**多选**:
```
选择必需的防护装备:
  ☑ 护目镜
  ☑ 实验服
  ☐ 手套
```

#### 4. 顺序型检查点

- **界面**: 可拖拽的列表
- **操作**: 拖动项目到正确位置

### 反馈系统

#### 通过反馈(绿色)
```
✓ 通过! 答案正确
```

#### 失败反馈(红色)
```
✗ 答案错误: 输入值超出允许范围
期望: 25.0±0.5 mL, 实际: 26.0 mL
```

### 导航

- **上一步**: 返回前一步骤(可重新操作)
- **提交**: 验证当前答案
- **下一步**: 进入下一步骤(需先提交通过)

### 进度追踪

- **进度条**: 显示完成百分比
- **步骤计数**: "当前: 3/10"
- **错误统计**: 实时显示错误次数

## 查看报告

### 生成报告

1. 完成所有步骤后,点击"完成实验"
2. 弹出最终得分对话框
3. 选择"生成报告"

### 报告内容

#### 1. 基本信息
- 实验名称
- 学生ID
- 开始/结束时间
- 用时

#### 2. 最终得分
- 大字显示总分
- 评分等级

#### 3. 实验摘要
- 总步骤数
- 通过步骤
- 总错误数

#### 4. 步骤详情
每个步骤显示:
- 标题和操作说明
- 通过/失败状态
- 得分
- 尝试次数
- 错误记录(如有)

### 导出报告

#### 导出HTML
```
文件 → 生成报告 → 导出HTML
```
- 可在浏览器中查看
- 支持打印

#### 导出PDF (需要安装 WeasyPrint)
```
文件 → 生成报告 → 导出PDF
```
- 适合存档和提交

## 常见问题

### Q1: 程序启动失败,提示缺少模块?

**A**: 请确保已安装所有依赖:
```bash
pip install -r requirements.txt
```

如果使用虚拟环境,确保已激活:
```bash
# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate
```

### Q2: 提交后没有反馈?

**A**: 可能的原因:
1. 未填写必填字段
2. 输入格式不正确(如输入文字而非数字)
3. 检查状态栏是否有错误提示

### Q3: 如何查看历史记录?

**A**: 目前版本需手动查看 `data/records/` 目录下的JSON文件。
未来版本将提供图形界面浏览功能。

### Q4: 实验进行到一半意外关闭,能恢复吗?

**A**: 当前版本不支持自动保存进度。建议一次性完成实验。
计划在未来版本添加自动保存功能。

### Q5: 曲线显示空白?

**A**: 可能原因:
1. PyQtGraph未安装: `pip install pyqtgraph`
2. 参数配置错误
3. 查看日志文件 `logs/app.log` 获取详细信息

### Q6: 如何添加自己的实验?

**A**:
1. 在 `assets/templates/` 下创建新的YAML文件
2. 参考现有模板编写
3. 使用验证工具检查:
   ```bash
   python scripts/validate_template.py assets/templates/my_exp.yaml
   ```
4. 重启程序,新实验会出现在列表中

### Q7: 报告导出失败?

**A**:
- **HTML导出**: 应该总是可用
- **PDF导出**: 需要安装 `weasyprint`
  ```bash
  pip install weasyprint
  ```

  Windows用户可能还需要安装 GTK+:
  https://github.com/tschoonj/GTK-for-Windows-Runtime-Environment-Installer

## 快捷键

### 通用
- `Ctrl+N`: 新建实验
- `Ctrl+R`: 重新开始
- `Ctrl+Q`: 退出程序
- `F1`: 帮助

### 实验操作
- `Enter`: 提交当前步骤
- `Ctrl+→`: 下一步
- `Ctrl+←`: 上一步
- `Esc`: 取消操作

### 报告
- `Ctrl+E`: 导出HTML报告
- `Ctrl+P`: 打印报告(如已打开报告)

## 使用技巧

### 1. 理解实验原理

- 仔细阅读每一步的说明
- 点击知识点链接查看详细信息
- 理解而非记忆答案

### 2. 注意安全提示

- 红色警告框表示危险操作
- 黄色提示框表示注意事项
- 务必按要求佩戴防护装备

### 3. 容差范围

- 数值输入通常有容差(如 ±0.5)
- 过于精确的值也可能不被接受
- 模拟真实实验的测量误差

### 4. 多选题

- 仔细查看是"选择所有"还是"选择最佳"
- 部分选择会被判定为错误

### 5. 提高得分

- 减少错误尝试次数
- 完成所有步骤
- 注意安全和规范操作
- 部分实验有额外奖励项

## 附录

### A. 实验难度说明

- **初级 (Beginner)**: 基础操作,10-30分钟
- **中级 (Intermediate)**: 综合实验,30-60分钟
- **高级 (Advanced)**: 复杂流程,60-120分钟

### B. 评分等级

- **90-100**: 优秀 (Excellent)
- **80-89**: 良好 (Good)
- **70-79**: 及格 (Pass)
- **<70**: 需改进 (Need Improvement)

### C. 支持的文件格式

- **模板**: YAML (.yaml, .yml)
- **报告**: HTML (.html), PDF (.pdf)
- **记录**: JSON (.json)
