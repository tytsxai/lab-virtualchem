## 📋 必做清单（按优先级排序）

### 🔴 严重（必须完成 - 30分钟）

#### 1. 生成并配置JWT密钥 ⭐⭐⭐

**为什么重要**: 默认密钥会导致认证系统完全失效

```bash
# 方法1: 使用自动工具
python scripts/pre_launch_fixes.py --generate-secrets

# 方法2: 手动生成
python -c "import secrets; print('JWT_SECRET_KEY=' + secrets.token_urlsafe(64))"
```

**配置方法**:

```bash
# 1. 查看生成的密钥
cat secrets.txt

# 2. 配置到环境变量（Linux/Mac）
export JWT_SECRET_KEY="生成的密钥"

# 或者配置到.env文件
echo "JWT_SECRET_KEY=生成的密钥" >> .env
```

**验证**:

```bash
# 启动应用，检查日志不应出现"使用默认JWT密钥"警告
python main.py
```

#### 2. 禁用生产环境的开发者模式 ⭐⭐⭐

**编辑文件**: `config/production.json`

```json
{
    "developer": {

        // ... 其他配置保持不变
    }
}
```

**或者使用自动工具**:

```bash
python scripts/pre_launch_fixes.py --fix-config
```

### 🟡 重要（强烈建议 - 1小时）

#### 4. 修复print语句 ⭐⭐

**扫描问题**:

```bash
python scripts/pre_launch_fixes.py --fix-prints
# 会生成 print_statements_fix_guide.md 文件
```

**修复策略**:

对于每个print语句，根据类型选择：

```python
# 1. 调试信息 → logger.debug()
# 之前:
print(f"开始处理: {data}")

# 之后:
logger.debug("开始处理", extra={"data": data})


# 2. 正常信息 → logger.info()
# 之前:
print("实验完成")

# 之后:
logger.info("实验完成")


# 3. 警告信息 → logger.warning()
# 之前:
print("警告: 温度过高")

# 之后:
logger.warning("温度过高", extra={"temperature": temp})


# 4. 错误信息 → logger.error()
# 之前:
print(f"错误: {e}")

# 之后:
logger.error("操作失败", exc_info=True)


# 5. 用户输出 → 保留或使用UI
# 命令行工具可以保留print
# GUI应用应使用状态栏或消息框
```

**批量替换（需要审查）**:

使用IDE的查找替换功能：

```
查找: print\((.*)\)
替换: logger.info(\1)
```

⚠️ **注意**: 需要手动审查每个替换，选择合适的日志级别

## 🆘 常见问题

### Q1: JWT密钥配置后仍提示默认密钥？

**A**: 检查环境变量优先级

```bash
# 优先级（从高到低）
1. 环境变量 JWT_SECRET_KEY
2. .env 文件
3. config.json

# 验证环境变量
echo $JWT_SECRET_KEY  # Linux/Mac
echo %JWT_SECRET_KEY%  # Windows

# 确保配置正确加载
python -c "import os; print(os.getenv('JWT_SECRET_KEY'))"
```

### Q2: 修复print后测试失败？

**A**: 部分测试可能依赖print输出

```python
# 解决方案1: 使用capsys捕获日志
def test_something(capsys):
    # 你的代码
    captured = capsys.readouterr()
    # 验证输出

# 解决方案2: mock logger
from unittest.mock import patch

@patch('src.utils.logger.get_logger')
def test_something(mock_logger):
    # 你的测试代码
```

### Q3: 打包后缺少文件？

**A**: 检查spec文件的数据文件配置

```python
# VirtualChemLab.spec
datas=[
    ('assets', 'assets'),
    ('config.json', '.'),
    ('assets/templates', 'assets/templates'),
]
```

### Q4: 性能测试失败？

**A**: 调整性能基准
