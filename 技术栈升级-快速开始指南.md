### 2. 核心技术学习

**PyMunk（物理引擎）**:
```python
# 快速示例
import pymunk

space = pymunk.Space()
space.gravity = (0, -981)  # 重力

# 创建刚体
body = pymunk.Body(1, 100)  # 质量1kg, 转动惯量100
shape = pymunk.Circle(body, 10)  # 半径10
space.add(body, shape)

# 物理步进
for i in range(60):
    space.step(1/60.0)
```

**Numba（JIT加速）**:
```python
# 快速示例
from numba import jit
import numpy as np

@jit(nopython=True)  # 添加这一行即可
def fast_calculation(arr):
    result = 0.0
    for i in range(len(arr)):
        result += arr[i] ** 2
    return result

# 自动加速50-100倍！
arr = np.random.rand(1000000)
result = fast_calculation(arr)
```

**SQLAlchemy（数据库）**:
```python
# 快速示例
from sqlalchemy import create_engine, Column, Integer, String
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker

Base = declarative_base()

class Experiment(Base):
    __tablename__ = 'experiments'
    id = Column(Integer, primary_key=True)
    name = Column(String(255))

# 创建数据库
engine = create_engine('sqlite:///data.db')
Base.metadata.create_all(engine)

# 使用
Session = sessionmaker(bind=engine)
session = Session()
session.add(Experiment(name='酸碱滴定'))
session.commit()
```

## 🛠️ 常见问题

### Q1: 必须全部升级吗？
**A**: 不必须。您可以选择性升级某些部分。建议优先级：
1. 打包配置（立即可用）
2. 物理引擎（性能提升最大）
3. Numba加速（实施简单，效果好）
4. 数据库迁移（需要数据迁移工作）

### Q2: 会影响现有功能吗？
**A**: 方案设计为向后兼容。所有升级都保持API兼容性，不会破坏现有功能。

### Q3: 需要多长时间？
**A**:
- **仅打包**: 1天（使用现有代码直接打包）
- **渐进优化**: 2-4周（选择性实施部分优化）
- **完整升级**: 2-3个月（实施所有优化）

### Q4: 打包后体积多大？
**A**:
- **开发环境**: ~500MB（包含所有依赖）
- **打包后**: ~120-150MB（使用UPX压缩）
- **安装程序**: ~80-100MB（Inno Setup/DMG压缩）

### Q5: 是否支持自动更新？
**A**: 是的。方案包含完整的自动更新系统，用户可以一键更新到最新版本。

### Q6: macOS代码签名必需吗？
**A**: 不必需，但强烈推荐：
- **不签名**: 用户需要手动允许运行（安全提示）
- **签名**: 无需额外操作，直接运行
- **公证**: 可以通过Gatekeeper，最佳用户体验

### Q7: 如何回退？
**A**: 升级前会提示您备份。如需回退：
```bash
git checkout v2.0.0
pip install -r requirements-old.txt
```
